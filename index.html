Lets Git Going!

This tutorial is for people who have a basic understanding of what Git is and want a better understanding of how to use Git in their work-flow. In order to continue with this tutorial you should:

  1. Download Git
  2. Have an account on GitHub
  3. Have downloaded a text editor (Atom/Sublime)

Now, let’s git started!

Git Solo- Personal projects and posting to GitHub

  1. Create an empty directory
  2. Add a file (.css, .js, .ejs) (you can open text editor to make changes if you want)
  3. git init
  4. git status
  5. git add file_name.html (or git add . to add all files)
  6. Git commit -m”do NOT forget to comment”
  7. Log onto your Github and make a new Github repository
  8. Copy remote/push from GitHub page + paste on your terminal.
  9. Refresh your GitHub page- should have ALL files loaded!

    10. Keep going...If you would like to make changed to your file and re-upload follow these next steps:

  10. Make changes on name.html ( add code or comments) on text editor and save files
  11. git add
  12. git commit -m" "
  13. git push -u origin master
  14. Refresh page + voila!

Git Group- working with an immediate team.

  1. Go to the Github master repository and clone, click clone and copy the URL
  2. On your terminal, cd into your desktop (cd into the folder you want to place your clone in) git clone URL then ls to check what is in there
  3. cd into your clone folder
  4. Git fetch (make sure to ALWAYS fetch before you edit!)
  5. Git merge
  6. Git branch branch_name (creating your own branch to make edits)
  7. Git checkout branch_name
  8. Open your text editor and start making edits
  9. Git add file_name
  10. Git status
  11. Git commit -m”do NOT forget to comment”
  12. Git push branch_name

    13. Warning! This section can get tricky. If someone pushes at the same time as you it will return as a merge conflict. No need to worry, you can now follow these steps:

  13. Git fetch (Note: do NOT pull, pull loses content)
  14. Git merge (this will show changes on your editor, delete any unwanted marks)

Git Fork

  1. Go to the Github repository that you’re interested in forking
  2. Click fork → this will essentially create a copy of the repository onto your profile where you can make changes and edits without it affects the original repository
  3. From the newly copied repository in your profile, click click and copy URL
  4. On your terminal, cd into your desktop (cd into the folder you want to place your clone in) git clone URL then ls to check what is in there
  5. Git branch branch_name (creating your own branch to make edits)
  6. Git checkout branch_name
  7. Open your text editor and start making edits
  8. Git add file_name
  9. Git status
  10. Git commit -m”do NOT forget to comment”
  11. Git merge master (or git checkout master then git merge ?)
  12. Git push -u origin master

    13. If you want to incorporate your changes into the original repository:

  13. Create a pull request
  14. The original owner will see the pull request and if they like it, they will merge the request.
